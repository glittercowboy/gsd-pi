# Phase 20.2 Plan 01: Headless Browser Mode + Screenshot Extraction Summary

**One-liner:** GSD_BROWSER_HEADLESS env var suppresses Chromium window popup; claude-process extracts base64 screenshots from browser_* tool results and emits browser_state_update events for relay to preview panel.

## What Was Built

### BrowserStateUpdate Type (chat-types.ts)
New exported interface at end of file:
```typescript
export interface BrowserStateUpdate {
  type: "browser_state_update";
  screenshot: string;  // base64-encoded JPEG
  url: string;
  title?: string;
  toolName?: string;   // which browser_* tool produced this
}
```

### Headless Mode Control (browser-tools/index.ts line 346)
Replaced hardcoded `{ headless: false }` with:
```typescript
const headless = process.env.GSD_BROWSER_HEADLESS === "1";
const launchOptions: Record<string, unknown> = { headless };
```
When `GSD_BROWSER_HEADLESS` is unset or `"0"`, browser opens visually (default). When `"1"`, headless mode suppresses the Chromium window.

### Env Injection (claude-process.ts)
After `delete env.CLAUDECODE`, added:
```typescript
env.GSD_BROWSER_HEADLESS = "1";
```
Mission Control now spawns GSD with headless browser mode always active.

### Screenshot Extraction (claude-process.ts)
- `currentToolName` variable tracks the active tool across events
- `tool_execution_start`: sets `currentToolName = toolName`
- `tool_execution_update`: after emitting text delta, scans `content` array for `{ type: "image", data: base64 }` items when tool name starts with `browser_`; if found, emits `browser_state_update` event with screenshot, URL, and title extracted via regex
- `tool_execution_end`: resets `currentToolName = ""`
