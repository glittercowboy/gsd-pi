<decisions_made>

- Use pi-coding-agent AuthStorage NOT custom Rust OAuth — already has registered client IDs,
  working token refresh, file locking, and 4+ providers
- Device code flow (URL + code paste) not redirect URI — avoids deep link complexity entirely
- Auth state lives in ~/.gsd/auth.json — single source of truth for both CLI and Mission Control
- tauri-plugin-deep-link commit stays (it's still correct if we ever need it), but OAuth flow
  will not use the gsd:// redirect URI at all
- Bun server is the auth broker — frontend never touches keychain directly
</decisions_made>
